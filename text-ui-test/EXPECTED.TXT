Ledger67 is a double-entry book-keeping CLI program that ensures your accounts are BALANCED!
Balanced means that the LEFT side = RIGHT side of this equation:
Assets = Equity - Liabilities + (Income - Expenses)
Valid Root Accounts: assets, liabilities, equity, income, expenses
Example: Assets:Cash or Income:Job
Enter Command: === Ledger67 Help ===
Available commands:

1. add - Add a new transaction
   Format (Manual): add -date DATE -desc DESCRIPTION -p POSTING1 -p POSTING2 -c CURRENCY
   ASSETS = EQUITY - LIABILITIES + (INCOME - EXPENSES)
   Each transaction must be balanced. This is checked by the system automatically.
   Postings support hierarchical account names using ':'
   Examples: Assets:Cash, Assets:Bank:DBS, Expenses:Food, Income:Salary
   Example: add -date 18/03/2026 -desc Office supplies -p "Assets:Cash -45.50" -p "Expenses:OfficeSupplies 45.50" -c SGD
   Format (Preset): add -date DATE -preset TYPE AMOUNT -c CURRENCY
   Presets: DAILYEXPENSE, INCOME, BUYINGSTOCKS
   Example: add -date 18/03/2026 -preset DAILYEXPENSE 50.00 -c SGD
   Example: add -date 18/03/2026 -desc Office supplies -p "Assets:Cash -45.50" -p "Expenses:OfficeSupplies 45.50" -c SGD

2. list - Display and filter transactions
   Format: list [-acc ACCOUNT] [-begin DATE] [-end DATE] [-match REGEX] [-to CURRENCY]
   - All flags are optional and can be combined (layered).
   - Dates must be in DD/MM/YYYY format.
   - Using -acc shows only postings for that account category.
   Example (Layered): list -acc Expenses -begin 01/01/2026 -end 31/01/2026
   Example (Regex):   list -match "(Lunch|Dinner)"

3. list -to CURRENCY - Display transactions with converted view values
   Format: list -to CURRENCY
   Example: list -to USD
   By default, this is a view-only feature.
   To store all displayed conversions, use: confirm all
   To store one displayed conversion, use: confirm ID
   Example: confirm 3

4. edit - Modify an existing transaction
   Format: edit ID [-date DATE] [-desc DESC] [-p POSTING] [-c CURRENCY]
   Example: edit 1 -desc Updated description -p "Expenses:Food 50" -p "Assets:Cash -50"

5. delete - Remove a transaction
   Format (Single): delete ID
   Format (Bulk):   delete [-begin DATE] [-end DATE] [-match REGEX] [-acc ACCOUNT]
   Note: Bulk deletion requires at least one filter flag.
   Example (Single): delete 3
   Example (Range):  delete -begin 01/01/2026 -end 15/01/2026
   Example (Match):  delete -match Steak

6. clear - Remove all transactions
   Format: clear

7. convert - Convert currencies
   Format: convert -a AMOUNT -from SOURCE_CURRENCY -to TARGET_CURRENCY
   Example: convert -a 100 -from USD -to SGD
   Note: This is a view-mode feature only. It does NOT create or store a transaction.

8. convert transaction - Convert an existing transaction
   Format: convert transaction ID -to TARGET_CURRENCY
   Example: convert transaction 3 -to SGD
   By default, this is a view-mode feature only & won't modify the stored transaction.
   To store the converted transaction, type: confirm

9. confirm - Store viewed converted transaction(s)
   After convert transaction ...
   Format: confirm

   After list -to ...
   Format A: confirm all
   Format B: confirm ID
   Example: confirm all
   Example: confirm 3

10. rates - Refresh live exchange rates
    Format: rates refresh

11. help - Show this help message
    Format: help

12. exit - Exit the application
    Format: exit

14. balance - Generate Balance Sheet
    Format: balance
    Optional A: balance -to CURRENCY
    Optional B: balance -acc ACCOUNT
    Optional C: balance -acc ACCOUNT -to CURRENCY
    Example: balance -to USD
    Example: balance -acc Assets:Bank

15. uiassist - Turn on/off UI Assistance
    Format: uiassist -on/off
    With UI Assistance on, the program will guide the user through using prompted inputs

=== Additional Information ===
- DATE format: DD/MM/YYYY (e.g., 18/03/2026)
- CURRENCY must be one of: SGD, USD, EUR
- ID is shown when using the 'list' command
- Transactions are stored locally on your machine
- Exchange rates are loaded from exchange-rates.json
- Converted values are view-only unless confirmed
- Any command other than confirm will ignore the pending conversion
- Account names may be hierarchical using ':'
- Example account names: Assets:Cash, Assets:Bank:DBS, Expenses:Food
- Valid root account types are: Assets, Liabilities, Equity, Income, Expenses

For detailed documentation, please refer to the User Guide.
